Content

This view shows a hierarchy of all religious denominations contained in the database. The hierarchy is represented by an indented tree visualization. The main religions are on the first level. For other religions beside Christianity, Islam and Judaism, we made the pragmatic choice to group them under a category of other religions. The main reason is that those are not the focus, and that this grouping allows symbols in the map view with no more than four subelements.

The religious denominations each have a distinct color which is stored in the database. The color hue is selected such that; for the three Abrahamic religions; red (Christianity), green (Islam), and blue (Judaism) hues are used. However, the difference between colors is limited because of the amount of denominations. In Confidence Mode, the nodes are colored based on the average confidence of the represented data. Beside the nodes, a shorthand for the religious denomination is written.

Interaction

Selection

When clicking a node (brushing), the node is highlighted. Subsequently, data from those religious denominations is highlighted (linked) throughout all other views. Non-brushed nodes are displayed darker than usual. In the same manner, brushing in any other view will link the relevant religions in the hierarchy.

Partial Highlighting

The color of the nodes is modified based on how much of the data for the node is filtered out. In only active mode, a node is entirely colorful if any data with that religious denomination is not filtered out, and less colorful otherwise. In all data mode, the node is split up into two areas, showing the ratio of filtered-out data. The amount of filtered-out data is represented by an area with lower saturation.

In Confidence Mode, each node is split up into more areas: One for each active confidence value, and one for each inactive confidence value. Inactive values again have a lower saturation. The aspect of confidence shown depends on which one is selected as visible in the confidence pane.

Filtering

Religions can be deselected, and the respective religious denominations are filtered out of the data throughout the entire visualization. Depending on whether the only active mode is activated, the filtered-out data is either hidden or greyed out. There are two basic modes for filtering by religion: In simple mode, evidence matching any of the checked religions is shown. In advanced mode, only evidence from cities where selected combinations of religions exist are shown. The filters are only applied to the dataset once you click the Apply button, which is enabled once there are changes to the filter.

Simple Mode

In simple mode, all data on any checked religion is filtered. Initially, all religions are selected. To filter out a religion, uncheck it in the checkbox column, and then press Apply.

Advanced Mode

In advanced mode, it is possible to create multiple groups of selected religions. In this case, the data is filtered grouped by place as follows: A piece of evidence is active if for any group, all contained religions are present in the place. For instance, if analysis requires to only show places where the Syriac Orthodox Church (SYR), the Armenian Church (ARM), and the Church of the East (COE) coexist, but not those where only one of the three is present:

  • Switch to advanced mode.
  • Create two additional columns by pressing the + plus button
  • In the first column, activate SYR and ARM.
  • In the second column, activate SYR and COE.
  • In the first column, activate ARM and COE.
  • Press apply.
Controls

In advanced mode, columns can be added after a column by using the + plus button above the column. Every column (except the last remaining one) can be deleted by pressing the × cross button above the column. The column's checkbox selection can be inverted using the ! exclamation sign button.
